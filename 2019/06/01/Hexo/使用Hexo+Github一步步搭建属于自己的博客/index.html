使用Hexo+Github一步步搭建属于自己的博客

       本文将使用Hexo+Github一步步搭建属于自己的博客，不多说，先上图

搭建步骤

  • 注意

    • 本文中所有命令或配置中以#开头的表示注释，仅仅用于说明

  1. 安装Node.js

    这里不对Node.js的安装做讲解，Node.js官网下载安装即可（https://nodejs.org/zh-cn/）

    安装配置好Node.js后可以通过以下命令查看node和npm的版本

  2. 安装Git

    这里不对Git的安装做讲解，可以参考https://git-scm.com/book/zh/v2/起步-安装-Git

    安装好后，通过以下命令可以查看git的版本

  3. 创建Github仓库

    • 打开github并登录（https://github.com/）

    • 新建仓库

      • 选中Repositories栏，然后点击右侧的New按钮创建

      • 设置仓库名字

        1. 仓库名必须严格遵守格式：用户名.github.io，不要随意起名，后续会有麻烦，这里vimior是我的用户名）
        2. 记得选中Public和勾选Initialize this repository with a README

    • 查看Github Pages

      • 选中刚刚创建的仓库的Settings栏，找到Github Pages的设置项

      • 此时Github Pages会显示如下，表明当前仓库可以作为Github Pages

  4. 安装Hexo环境

    • 安装Hexo

    • 新建一个工程目录（这里是vimior_blog），然后进入这个目录

    • 初始化Hexo

    • 生成静态页面

    • 启动本地服务器，启动后可以通过http://localhost:4000来访问

    • 新建博文

      通过hexo new <文章名>来新建或者直接在工程目录下的source/_post目录下创建一个md文件即可，然后重新生成静态页面并启动服务器即可

      博文格式如下

  5. 配置Hexo

    注意：这里我们先统一把需要的包安装好，具体配置会在配置里面用注释说明

    • 安装hexo-theme-next主题（配置使用在后面统一说明）

    • 安装hexo-helper-live2d（用于生成Live2D看板娘）

    • 安装hexo-symbols-count-time（用于字数统计和计算阅读时间）

    • 安装hexo-generator-sitemap（用于生成站点地图）

    • 安装hexo-generator-feed（用于生成RSS订阅）

    • 安装hexo-generator-searchdb（用于生成搜索数据库）

    • 安装hexo-deployer-git（用于推送到Github）

    • 新建categories页（分类页面）

    • 新建tags页（标签页面）

    • 新建about页

    • 修改工程的配置修改_config.yml文件）

      详细配置参考https://hexo.io/zh-cn/docs/configuration

    • 修改hexo-theme-next主题的配置（修改themes/next/_config.yml__文件）

      详细配置参考http://theme-next.iissnan.com/theme-settings.html

  6. 推送到Github Pages

    注意：

    1. 推送前先本地运行，通过浏览器访问没问题再行推送

    2. 初次使用git需要设置user name和email

    3. （如果未曾添加过ssh-agent）需要添加ssh-key到github的ssh-agent



    • 生成静态文件

    • 推送到github pages

总结

      此时，我们已经搭建好一个属于自己的博客了，如果需要部署到其他仓库，只需要修改deploy配置即可，详情请查阅https://hexo.io/zh-cn/docs/deployment

      Hexo的功能很强大，用 Hexo+Github搭建一个属于自己的博客，不需要我们写代码，只需要通过修改配置即可搭建一个属于自己的酷炫博客。另外，我们平时可以通过在source/_post目录下用Markdown来写自己的笔记、博文，然后重新推送即可在自己的线上博客上看到。
